# Product Docs

本目录维护产品层事实：为什么做、给谁用、解决什么问题、当前做什么、不做什么。实现细节放到 `docs/tech/`，测试方案放到 `docs/test/`。

## 文档索引

| 文档 | 维护内容 |
|---|---|
| [prd.md](prd.md) | 当前产品需求、功能范围、用户故事、非目标和后续需求池 |
| [roadmap.md](roadmap.md) | 阶段目标、里程碑、优先级和交付节奏 |

## 维护规则

1. 产品需求新增、变更、延期或删除时，先更新 [prd.md](prd.md)。
2. 需求影响阶段节奏时，同步更新 [roadmap.md](roadmap.md)。
3. 不在本目录记录具体代码方案、数据库表结构或测试日志。
4. 对 agent 来说，产品文档是实现边界，不是建议列表。
