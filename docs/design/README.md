# Design Docs

本目录维护设计与 UI 层事实：体验目标、信息架构、用户流、交互规则、组件状态、视觉规范和可用性风险。产品范围放到 `docs/product/`，实现方案放到 `docs/tech/`，验证证据放到 `docs/verification/`。

## 文档索引

| 文档 | 维护内容 |
|---|---|
| [ui-guidelines.md](ui-guidelines.md) | UI 设计原则、布局、交互、状态、文案和可访问性规则 |
| [design-system.md](design-system.md) | 组件、颜色、排版、间距、图标、动效和响应式约束 |

## 维护规则

1. 设计、交互、视觉、组件状态或可访问性变化，更新本目录。
2. 产品需求只写用户价值和范围，不把 UI 细节塞进 PRD。
3. 技术实现只写架构和代码边界，不把视觉规范塞进技术设计。
4. 设计验收证据、截图、走查结论和阻塞点写到 `docs/verification/`。
