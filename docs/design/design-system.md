# Design System

本文档维护组件、视觉和交互规范。当前是初版占位，后续绑定具体应用后再补充真实 token 和组件。

## 组件分类

| 类型 | 用途 | 文档要求 |
|---|---|---|
| 导航 | 页面切换、目录跳转、任务入口 | 说明层级、选中态、禁用态 |
| 表单 | 输入、选择、提交 | 说明校验、错误、空值、加载态 |
| 列表 / 表格 | 信息扫描、比较、批量操作 | 说明密度、排序、筛选、空状态 |
| 状态反馈 | 成功、失败、加载、阻塞 | 说明触发条件和恢复路径 |
| 命令入口 | 保存、运行、验证、导出等操作 | 说明是否可逆、是否需要确认 |

## 维护要求

1. 组件规则必须写清楚状态，不只写静态样式。
2. 视觉 token 一旦被实现引用，就不能只在代码里改，必须同步更新本文档。
3. 新增复杂交互时，需要在 `docs/verification/` 补对应设计走查或可用性验证场景。
