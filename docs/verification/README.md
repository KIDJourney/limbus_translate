# Verification Docs

本目录维护验证层事实：不同业务场景的验证方式、执行证据、通过/失败结论、阻塞点、未覆盖项和回滚说明。测试策略和测试用例放到 `docs/test/`；本目录关注“这件事是否真的在目标场景中有效”。

## 文档索引

| 文档 | 维护内容 |
|---|---|
| [scenarios.md](scenarios.md) | 按业务场景组织的验证方式和通过标准 |
| [methods.md](methods.md) | 自动化、手动、设计走查、E2E、发布、回滚等验证方法 |
| [verification-log.md](verification-log.md) | 最近验证记录、命令输出摘要、证据、风险和未覆盖项 |

## 维护规则

1. 每个业务场景要有独立的验证方法，不能用单一命令替代所有场景。
2. 验证结论必须写清楚时间、环境、步骤、证据、结果和覆盖边界。
3. 无法验证时，不写“通过”，要记录阻塞点、已覆盖范围和剩余风险。
4. 业务场景变化时，同步更新 [scenarios.md](scenarios.md)。
5. 验证方法变化时，同步更新 [methods.md](methods.md)。
