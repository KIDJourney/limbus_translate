# Test Docs

本目录维护测试层事实：测试策略、测试方案、测试用例、自动化覆盖和回归要求。业务场景验证方法、证据和结论放到 `docs/verification/`。

## 文档索引

| 文档 | 维护内容 |
|---|---|
| [test-plan.md](test-plan.md) | 当前测试策略、覆盖范围、验收标准和回归要求 |

## 维护规则

1. 每个可交付变更都要能映射到测试方案或验收标准。
2. 测试方案要说明覆盖范围、执行方式、通过标准和不覆盖范围。
3. 真实执行后的证据、结果和阻塞点写到 `docs/verification/`。
4. 回归问题修复后，补充对应测试或手动验收步骤。
